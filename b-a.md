# TBeye UI/UX 改善レポート

## 概要

TBeye UI は、介護施設向けの通話管理・AI 議事録システムです。本プロジェクトでは、ニールセンのユーザビリティ 10 原則に基づいて UI を分析し、段階的に改善を実施しました。

---

## 改善前の課題

| 問題カテゴリ             | 内容                                               |
| ------------------------ | -------------------------------------------------- |
| **情報過多**             | 録音オプションがデフォルトで全展開、スクロール過剰 |
| **一貫性の欠如**         | ページごとに異なる検索 UI、カードスタイルの混在    |
| **視覚的階層の不明確さ** | 重要なアクションが他の要素に埋もれている           |
| **認知負荷が高い**       | 多数のトグル・選択肢が常に表示、絵文字多い         |

---

## 設計原則

本改善では以下の原則に基づいて実装しました：

1. **一画面一目的** - 各ページで最も重要なタスクを 1 つに絞る
2. **プログレッシブ・ディスクロージャー** - 詳細は必要時に表示
3. **3 クリックルール** - 主要タスクは 3 クリック以内で完了
4. **余白を恐れない** - 情報密度を適切に下げる
