# TBeye UI/UX リデザイン調査レポート

## 1. UXデザインの重要原則（2025年版）

### 1.1 ニールセンのユーザビリティ10原則

| # | 原則 | 説明 |
|---|------|------|
| 1 | システム状態の可視性 | ユーザーに現在の状態を常にフィードバック |
| 2 | システムと現実世界の一致 | ユーザーが理解できる言葉・概念を使用 |
| 3 | ユーザーの制御と自由 | 間違いから簡単に戻れる「非常口」を用意 |
| 4 | 一貫性と標準 | 同じ操作は同じ結果を生む |
| 5 | エラーの予防 | そもそもエラーが起きにくい設計 |
| 6 | 記憶より認識 | 見てわかるデザイン、記憶に頼らせない |
| 7 | 柔軟性と効率性 | 初心者にも上級者にも使いやすい |
| 8 | 美的でミニマルなデザイン | 不要な情報を削減 |
| 9 | エラーからの回復支援 | エラー時の明確なガイダンス |
| 10 | ヘルプとドキュメント | 必要時にすぐアクセス可能 |

**出典**: [Nielsen Norman Group - 10 Usability Heuristics](https://www.nngroup.com/articles/ten-usability-heuristics/)

### 1.2 2025年のUXトレンド

- **シンプルさの追求**: 情報過多を避け、本質的な機能に集中
- **アクセシビリティ最優先**: 後付けではなく設計段階から考慮
- **認知負荷の軽減**: 一度に表示する情報量を制限（マジックナンバー7±2）
- **一貫性**: ボタン、フォント、色の統一

**出典**:
- [UXPin - UX Design Principles](https://www.uxpin.com/studio/blog/ux-design-principles/)
- [Laws of UX](https://lawsofux.com/)

## 2. 現在のプロダクト分析

### 2.1 ページ構成

| ページ | 目的 | 主要ユーザータスク |
|--------|------|-------------------|
| ai-minutes.html | AI議事録 | 録音開始、文字起こし確認、要約編集 |
| sip-groups.html | SIP監視 | 16グループの状態確認 |
| realtime.html | 通話状況 | グループ詳細、子機状態確認 |
| call-logs.html | 通話履歴 | 過去の記録検索・閲覧 |
| staff.html | ユーザー管理 | ユーザーの追加・編集・権限管理 |
| dictionary.html | 辞書・用語 | 用語の登録・編集・削除 |

### 2.2 発見した問題点

#### A. 情報過多（原則8違反）

| 問題 | 該当ページ | 重要度 |
|------|-----------|--------|
| 録音オプションがデフォルトで展開 | ai-minutes.html | 高 |
| 16グループを一度に表示 | sip-groups.html | 中 |
| 複数セクションが同時に表示 | ai-minutes.html | 高 |
| ページ内のカード・バッジ・ステータスが多すぎ | 全ページ | 高 |

#### B. 視覚的階層の不明確さ

| 問題 | 詳細 |
|------|------|
| プライマリアクションの埋没 | 重要なボタンが他の要素に埋もれている |
| 絵文字への過度な依存 | 意味が不明確、一貫性がない |
| ボタンスタイルの乱立 | btn-primary, btn-outline, btn-ghost, btn-sm, btn-lgなど多すぎ |

#### C. 一貫性の問題（原則4違反）

| 問題 | 例 |
|------|-----|
| ページタイトルの形式 | 絵文字あり/なしが混在 |
| カードのスタイル | item-card, status-card, device-cardなど統一されていない |
| 各ページで独自の`<style>`ブロック | 重複コード、保守性低下 |

#### D. 認知負荷が高い

- ai-minutes.htmlの録音オプションに多数のトグル・選択肢
- クイック辞書登録フォームが常に表示
- グループ選択で16個のオプションを一度に表示

#### E. アクセシビリティ

| 問題 | 影響 |
|------|------|
| 絵文字をアイコン代わりに使用 | スクリーンリーダーで意味が伝わらない可能性 |
| フォーカス状態が不明確 | キーボード操作が困難 |
| コントラスト比 | 一部テキストで不十分な可能性 |

## 3. リデザイン提案

### 3.1 設計原則（このプロダクト向け）

```
1. 一画面一目的: 各ページで最も重要なタスクを1つに絞る
2. プログレッシブ・ディスクロージャー: 詳細は必要時に表示
3. 3クリックルール: 主要タスクは3クリック以内で完了
4. 余白を恐れない: 情報密度を下げる
5. 絵文字からSVGアイコンへ: 一貫性とアクセシビリティ向上
```

### 3.2 具体的な改善案

#### A. AI議事録ページ（ai-minutes.html）

**現状の問題**:
- 録音ボタン + オプション + 文字起こし + 要約 + 履歴が全て表示
- ユーザーの主目的（録音開始）が埋もれている

**改善案**:
```
[録音前]
┌─────────────────────────────────┐
│  🎙️ 録音を開始                  │
│  ┌─────────────────────────────┐│
│  │     ▶ 録音を開始する        ││  ← 大きく目立つボタン
│  └─────────────────────────────┘│
│  ⚙️ 設定を変更                  │  ← 折りたたみ（デフォルト閉じ）
└─────────────────────────────────┘

[録音中]
┌─────────────────────────────────┐
│  🔴 録音中  00:02:34            │  ← 明確なステータス
│  ┌─────────────────────────────┐│
│  │     ⏹ 録音を停止            ││
│  └─────────────────────────────┘│
│  文字起こしがリアルタイムで表示  │
└─────────────────────────────────┘
```

#### B. SIP監視ページ（sip-groups.html）

**現状の問題**:
- 16グループを一度に表示
- 情報が多く、重要なステータスが埋もれる

**改善案**:
```
┌─────────────────────────────────────┐
│ 📊 概要                             │
│ アクティブ: 2  オンライン: 6  オフライン: 8 │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ ⚠️ 要注意（2件）                    │  ← 問題のあるものを先に
│ ├─ G5: オフライン 1時間             │
│ └─ G7: オフライン 30分              │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ ✅ 正常稼働（14件）                 │  ← 折りたたみ可能
└─────────────────────────────────────┘
```

#### C. 共通コンポーネントの統一

**ボタン**: 3種類に限定
- `btn-primary`: 主要アクション（1ページに1つ）
- `btn-secondary`: 二次的アクション
- `btn-text`: テキストリンク風

**カード**: 2種類に統一
- `card`: 標準カード
- `card-interactive`: クリック可能なカード

**アイコン**: 絵文字からSVGへ移行
```html
<!-- Before -->
<span class="nav-icon">🎙️</span>

<!-- After -->
<svg class="icon" aria-hidden="true">
  <use href="#icon-microphone"></use>
</svg>
```

### 3.3 カラーパレットの簡素化

**現状**: 多数のセマンティックカラー
```css
--primary, --success, --danger, --warning, --info, --neutral
```

**提案**: 4色に限定
```css
--color-brand: #2563eb;     /* ブランドカラー・主要アクション */
--color-positive: #059669;  /* 成功・正常 */
--color-negative: #dc2626;  /* エラー・危険 */
--color-neutral: #64748b;   /* 非アクティブ・補助 */
```

### 3.4 タイポグラフィの簡素化

**現状**: 8段階のフォントサイズ
```css
--text-xs, --text-sm, --text-base, --text-lg, --text-xl, --text-2xl, --text-3xl
```

**提案**: 4段階に限定
```css
--text-sm: 14px;    /* 補助テキスト */
--text-base: 16px;  /* 本文 */
--text-lg: 20px;    /* セクションタイトル */
--text-xl: 28px;    /* ページタイトル */
```

## 4. 優先順位付き実装ロードマップ

### Phase 1: クイックウィン（影響大・工数小）

| # | タスク | 対象 | 効果 |
|---|--------|------|------|
| 1 | 録音オプションをデフォルトで閉じる | ai-minutes.html | 認知負荷大幅減 |
| 2 | ページタイトルから絵文字を削除 | 全ページ | 一貫性向上 |
| 3 | 不要なバッジ・ステータス表示を削減 | 全ページ | 視覚的ノイズ削減 |
| 4 | ボタンスタイルを3種類に統一 | style.css | 保守性向上 |

### Phase 2: 構造改善（影響大・工数中）

| # | タスク | 対象 | 効果 |
|---|--------|------|------|
| 5 | SIPグループを「要注意」「正常」でグルーピング | sip-groups.html | 重要情報の発見性向上 |
| 6 | カードコンポーネントの統一 | style.css + 全HTML | 一貫性向上 |
| 7 | 余白の見直し（密度を下げる） | style.css | 可読性向上 |

### Phase 3: 本格リデザイン（影響大・工数大）

| # | タスク | 対象 | 効果 |
|---|--------|------|------|
| 8 | 絵文字をSVGアイコンに置換 | 全ページ | アクセシビリティ向上 |
| 9 | カラーパレット簡素化 | style.css | 視覚的一貫性 |
| 10 | モバイルファーストの再設計 | 全体 | モバイルUX向上 |
